 # 欢迎进入 Tu编程语言 的世界 

![从事创造性工作的人的插图。](./images/welcome/welcome-to-nft-school.svg)

欢迎来到另一种编程语言的描述。
与你以前学过的大多数语言不同，`Tu`并非真正意义上的通用语言，它是为解决面向数据且特定领域而准备的，例如：

>1. 如何编写一个高效利用CPU缓存的函数
>2. 如何使用图形来辅助理解函数的运作方式
>3. 在大数据上利用并行运算Tu所构建的复杂模型

在加快运算性能的同时也为理解逻辑使用了丰富且实用的图形化工具，让复杂模型相较于往常的编写体验变得更加易于使用。

>1. 可视化Graph编辑器，在逻辑冲突、异常时向用户询问，来达到更加稳定且即时易用的逻辑修改
>2. 可视化数据源编辑器，其主要功能在于分析逻辑通路来给与用户生成随机测试数据的建议
>3. 可视化性能分析器，提供运算性能、内存空间等分析，且配合即时修改达到一次运行调整至负载均衡最佳状态

为达到上述功能，我们一直面临着挑战，如如何分析和优化指令码得到我们所需要的信息，为此我们参考了`GCC`/`LLVM`等流行工具，而开发了专用的编译前端与编译后端，它使用了一套新的`中间码`并遵循着我们后端优化器最优解输出生成最终文件。输出除了`二进制`外，还可以生成`C`、`C#`、`Java`、`JavaScript`、`Python`、`OpenCL`的单文件源码来达到嵌入其他项目的目的。例如，如果你发明了一种新的压缩格式，你可以在`Tu`中实现它，并让其生成目标输出。在原项目中添加该输出源码文件，或生成`二进制`输出直接使用。

## 设计原则

1. Tu是一种为在几种语言中具有丰富经验的程序员设计的语言，是可扩展的，可以简单和其他语言项目进行融合。
2. Tu遵循[最小意外原则(POLA)](https://en.wikipedia.org/wiki/Principle_of_least_astonishment)，语法类似于F#，Tu不是F#的克隆，也不是它的子集，其差异来源主要受限于我们的输出目标过于庞杂，我们需要找到语言中的最小关联来设计`Tu`的特性。
3. Tu是面向数据的，虽然大多数的目标语言是面向对象的，但是我们设计之初`Tu`并非一个通用语言，它更类似于`Python`那样的粘合剂，但比`Python`而言它更适用于粘连`函数`而非`工具`，同时又有着自己与众不同且强大的能力，作者在近15年的编程经验中，发现解耦后的大部分模块，其重中之重就是设计一个高效且易于理解的数据模型来继续降低单个函数的复杂度，来让整体项目变得可持续发展。

## 输出

在`二进制`输出中编译器将会把Tu的逻辑编译成机器码外，还包含一个最小的运行时，一对`.c`/`.h`文件，包含可移植的、人类可读的C99代码，其他目标则会翻译成目标输出语言。

1. 如果`Tu`代码使用正则表达式、`List`、`Stack`、`HashSet`、`Dictionary`或`SortedDictionary`。
   C的输出依赖于[GLib](https://wiki.gnome.org/Projects/GLib)的实现。

2. `Math.FusedMultiplyAdd`在Python中的实现是用[pyfma](https://pypi.org/project/pyfma/)实现。

在没有GC的输出目标中(如C和C++)，对象和数组会被分配到堆栈上以获得最大的性能，或者在堆上获得额外的灵活性。
